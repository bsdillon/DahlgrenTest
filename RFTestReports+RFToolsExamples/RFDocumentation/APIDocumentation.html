Setting up the API

This documentation will explain how to set up a Data Charting API.


Importing Libraries

First, there are several JavaScript libraries that you will need to import into your code:

Chart.min.js
piechartlibrary.js
linechartlibrary.js
scatterchartLibrary.js
datacharting.js
chartstyle.js

 All of these libraries can be found in the DahlgrenTest repository in DahlgrenTest/RFTestReports+RFToolsExamples/RFDemoGUI and can be included as a source script in your code.

Example:

< script src="Chart.min.js" >< /script >
< script src="piechartlibrary.js" >< /script >
 etc...

Creating Graphs

The graphs in the Data Charting API are created using Chart.js and JSON objects. Chart.js is an open source JavaScript library used for data visualization and creating graphs. First, a new DataCharting object is created defining the size of the new graphs. Then, a JSON object is created containing the parameters for each graph that will be shown. Lastly, the JSON object and graph name are passed into the createChart() function for it to be built. The best way to have the graph displayed on screen is to append it to a table.

Example:

var data1 = [[100, 90, 80, 70], [10, 20, 40 ,50], [79, 34, 887, 234],...];
var labels = ['A','B','C',...];
var field = document.getElementById("field");
var table = document.createElement("TABLE");
field.appendChild(table);

charting = new DataCharting(300, 300);
var row = table.insertRow();
let td = document.createElement("td");
row.append(td);
var json = {"VAR_Example": {'type': chartType, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data1[1], 'modes': ['mode'], 'colorScheme':
[(RGB values)]}};
td.append(charting.createChart(json, "VAR_Example"));

Graph Components

The different parameters for a graph, including those shown in the example above, are:

  • type
  • x-axis
  • y-axis
  • series
  • x-data
  • y-data
  • modes
  • colorScheme
  • group

Type

The type parameter refers to the chart type of the graph. The different types consist of line graph, bar graph (vertical and horizontal), area graph, pie graph and scatter plot.

Line Graph Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data1[1], 'modes': [], 'colorScheme':
['rgb(127,240,52)','rgb(64,0,174)']}};
charting.createChart(json,"VAR_Example");


Vertical Bar Graph Example:

X-Axis/Y-Axis

The x-axis and y-axis parameters define the names of the x and y-axes, being whatever string the user specifies. However, in the case of a horizontal bar graph, the x and y-axes are reversed and therefore so are the axis labels. For pie charts the x and y-axis lables do not show up on the graph at all.

Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'VBar', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data1[1], 'modes': [], 'colorScheme':
['rgb(127,240,52)','rgb(64,0,174)']}};
charting.createChart(json,"VAR_Example");


Horizontal Bar Graph Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'HBar', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data1[1], 'modes': [], 'colorScheme':
['rgb(127,240,52)','rgb(64,0,174)']}};
charting.createChart(json,"VAR_Example");


Pie Chart Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example-0': {'type': 'Pie', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data1[1], 'modes': [], 'colorScheme': []}}; charting.createChart(json,"VAR_Example-0");


Series

The series parameter allows the user to distinguish between different data sets displayed on a graph by giving them a string label.

Example:

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': labels, 'y-data': data1[1], 'modes': ['Multi']},
"VAR_Example-1": {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 2', 'x-data': labels, 'y-data': data2[1], 'modes': ['Multi']},
"VAR_Example-2": {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 3', 'x-data': labels, 'y-data': data3[1], 'modes': ['Multi']},
"VAR_Example-3": {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 4', 'x-data': labels, 'y-data': data4[1], 'modes': ['Multi']}};
charting.createChart(json,"VAR_Example-0");

X-Data/Y-Data

The x-data and y-data parameters define the data that is used for the x and y-axes. Both parameters require either a set of arrays containing numeric data, or an array containing string data. Both axes can have numeric data and the x-axis can represent string data, however, the y-axis is only able to represent numeric data.

Numeric Data Example:

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': data2[1], 'y-data': data1[1], 'modes': ['Multi']},
"VAR_Example-1": {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': data2[1], 'y-data': data3[1], 'modes': ['Multi']},
"VAR_Example-2": {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': data2[1], 'y-data': data4[1], 'modes': ['Multi']},
"VAR_Example-3": {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': data2[1], 'y-data': data2[1], 'modes': ['Multi']}};
charting.createChart(json,"VAR_Example-0");

Numeric & String Data Example:

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': letters, 'y-data': numbers1, 'modes': ['Multi', 'Stacked']},
"VAR_Example-1": {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 2', 'x-data': letters, 'y-data': numbers2, 'modes': ['Multi', 'Stacked']},
"VAR_Example-2": {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 3', 'x-data': letters, 'y-data': numbers3, 'modes': ['Multi', 'Stacked']},
"VAR_Example-3": {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 4', 'x-data': letters, 'y-data': numbers4, 'modes': ['Multi','Stacked']}};
charting.createChart(json,"VAR_Example-0");

Modes

The modes parameter specifies which mode the graph will be displayed in.

If nothing is passed in for this parameter then the graph will default to showing only one data sample. However, if 'Multi' is passed in then multiple samples can be displayed on the graph; as many as there are specified in the JSON object array.

Single Vs. Multi Sample Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Line', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data, 'modes': [] }};
charting.createChart(json,"VAR_Example");

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': Line, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': labels, 'y-data': data1, 'modes': ['Multi']},
"VAR_Example-1": {'type': Line, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 2', 'x-data': labels, 'y-data': data2, 'modes': ['Multi']},
"VAR_Example-2": {'type': Line, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 3', 'x-data': labels, 'y-data': data3, 'modes': ['Multi']},
"VAR_Example-3": {'type': Line, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 4', 'x-data': labels, 'y-data': data4, 'modes': ['Multi']}};
charting.createChart(json,"VAR_Example-0");

If 'Mulit' and 'Stacked' are passed in, then the graph can still display multiple samples, however, the samples will be displayed stacked vertically rather than horizontally next to each other.

Multi vs. Multi Stacked Example:

var charting = new DataCharting(300, 300);

"VAR_Example-3": {'type': VBar, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 4', 'x-data': labels, 'y-data': data4, 'modes': ['Multi']}};
charting.createChart(json,"VAR_Example-0");

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': VBar, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': labels, 'y-data': data1, 'modes': ['Multi','Stacked']},
"VAR_Example-1": {'type': VBar, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 2', 'x-data': labels, 'y-data': data2, 'modes': ['Multi','Stacked']},
"VAR_Example-2": {'type': VBar, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 3', 'x-data': labels, 'y-data': data3, 'modes': ['Multi','Stacked']},

If 'Progressive' is passed in, then the graph will display each individual data point taken as an incremental value that is tracked over the entire graph. And if 'Discrete' or no specification is passed in, then the graph will show all of the data points plotted in no specific trend.

Discrete Vs. Progressive Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data 'modes': ['Discrete']}};
charting.createChart(json,"VAR_Example");

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Area', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data, 'modes': ['Progressive']}};
charting.createChart(json,"VAR_Example");

Lastly, if 'Trend' is passed in to the mode parameter on a scatter plot, then the graph will be displayed with a trend line.

Scatter Plot Vs. Scatter Plot Trend Line Example:

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data, 'modes': []}};
charting.createChart(json,"VAR_Example");

var charting = new DataCharting(300, 300);
var json = {'VAR_Example': {'type': 'Scatter', 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample', 'x-data': labels, 'y-data': data, 'modes': ['Trend']}};
charting.createChart(json,"VAR_Example");


Color Scheme

colorScheme sets the color of the samples displayed on the graph. If this parameter is not specified, then Chart.js will default the chart to a rainbow color scheme. However, the user can also designate their own color scheme to a graph by passing in the RGB values of the colors they wish to use.

var json = {"VAR_Example-0": {'type': 'Scatter', 'x-axis': 'X', 'y-axis': 'Y', 'series': 'sample 1', 'x-data': data2, 'y-data': data1, 'modes': ['Multi'], 'colorScheme': []},
"VAR_Example-1": {'type': 'Scatter', 'x-axis': 'X', 'y-axis': 'Y', 'series': 'sample 1', 'x-data': data2, 'y-data': data3, 'modes': ['Multi'], 'colorScheme': []},

"VAR_Example-3": {'type': 'Scatter', 'x-axis': 'X', 'y-axis': 'Y', 'series': 'sample 1', 'x-data': data2, 'y-data': data2, 'modes': ['Multi'], 'colorScheme': []}},
charting.createChart(json,"VAR_Example-0");

Group

Lastly, 'group' can be used to define the color scheme of a graph by passing in a group number.

Basic Color Scheme Vs. Group Example:

var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': 'Pie', 'x-axis': 'X', 'y-axis': 'Y', 'series': 'sample 1', 'x-data': labels, 'y-data': data5, 'modes': [], 'group':}};
charting.createChart(json,"VAR_Example-0");

var groupNumber = Math.floor(Math.random()*6);
var charting = new DataCharting(300, 300);
var json = {"VAR_Example-0": {'type': 'Pie', 'x-axis': 'X', 'y-axis': 'Y', 'series': 'sample 1', 'x-data': labels, 'y-data': data5, 'modes': [], 'group': groupNumber}}
charting.createChart(json,"VAR_Example-0");
