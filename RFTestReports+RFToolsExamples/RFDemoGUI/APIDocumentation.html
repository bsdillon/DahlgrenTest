Setting up the API

This documentation will explain how to set up a Data Charting API.


Importing Libraries

First, there are several JavaScript libraries that you will need to import into your code:

Chart.min.js
piechartlibrary.js
linechartlibrary.js
scatterchartlibrary.js
datacharting.js
chartStyle.js

 All of these libraries can be found in the DahlgrenTest repository in DahlgrenTest/RFTestReports+RFToolsExamples/RFDemoGUI and can be included as a source script in your code.

Example:
< script src="Chart.min.js" >< /script >
< script src="piechartlibrary.js" >< /script >
 etc...

Creating Graphs

The graphs in the Data Charting API are created using Chart.js and JSON objects. Chart.js is an open source JavaScript library used for data visualization. A JSON array object is created containing the parameters for each graph that will be shown. It is then passed into the createChart() function to actually create the graph.

Example:
charting = new DataCharting(300, 300);

var json = {"VAR_Example-0": {'type': chartType, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 1', 'x-data': labels, 'y-data': data1[1], 'modes': ['Multi']},
"VAR_Example-1": {'type': chartType, 'x-axis': 'time', 'y-axis': 'position', 'series': 'sample 2', 'x-data': labels, 'y-data': data2[1], 'modes': ['Multi']}};

charting.createChart(json, "VAR_Example-0");

Graph Components

The different parameters for a graph, including those shown in the example above, are:

  • type
  • x-axis
  • y-axis
  • series
  • x-data
  • y-data
  • modes
  • colorScheme
  • group
The type parameter refers to the chart type of the graph. The different types consist of line graph, bar graph (vertical and horizontal), area graph, pie graph and scatter plot. The x-axis and y-axis parameters speficy the names of the x and y-axes, the most common being time and position respectively. The series parameter specifies which sample the parameters are affecting. The x-data and y-data parameters define whether or not the data on the x and y-axes are string labels or numeric data. Both axes are able to be numeric values, however, only one axis is able to have string labels at one time. Finally, the modes parameter specifies which mode the graph will be displayed in. If 'Multi' is passed in for this parameter, then multiple samples can be displayed on the graph; as many as there are specified in the JSON object array. If 'Mulit' and 'Stacked' are passed in, then the graph can still display multiple samples, however, the samples will be displayed stacked vertically rather than horizontally next to each other. If 'Progressive' is passed in, then the graph will display more so the overall trend of the data. Lastly, if 'Discrete' is passed in, then the graph will show all of the data points plotted in no specific trend. colorScheme sets the color of the each sample displayed on the graph. If this parameter is not specified, then Chart.js will default the chart to a gray color scheme. Lastly, 'group' can be used to group samples together into a specified color scheme.
