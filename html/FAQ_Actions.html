Positive/negative paths
In a Workflow you can define each step to either execute on success, failure, etc. The default is to execute on the step only if it has been successful. These cases can be used to create error paths and to execute a set of tests that may not all pass. More information
- name: Mismatch answer math test
  if: ${{success() || failure() }}
  uses: ./.github/actions/mathTest
